# 250. Documentation dependency

Document లేకపోతే context lost అవుతుంది.
Without documentation, context gets lost.

ఇది ఎక్కడ documented?
Where is this documented?

Confluence link share చేయగలరా?
Can you share the Confluence link?

Document update అయ్యిందా last change తరువాత?
Has the doc been updated after the last change?

Outdated document follow చేస్తే issues వస్తాయి.
Following an outdated doc creates issues.

Latest version ఏది అని clearly mention చేయాలి.
We need to clearly mention which is the latest version.

Person dependency తగ్గించాలి documentation తో.
We need to reduce person dependency through documentation.

ఒక్కడే తెలిసిన knowledge risk.
Knowledge known to only one person is a risk.

ఆ person leave అయితే team stuck.
If that person leaves, the team is stuck.

Knowledge transfer documentation లో capture చేయాలి.
Knowledge transfer should be captured in docs.

Tribal knowledge ఎక్కువ ఉంది ఈ team లో.
There's a lot of tribal knowledge in this team.

అది document లోకి move చేయాలి.
We need to move that into documentation.

Runbooks update చేస్తున్నారా regularly?
Are runbooks being updated regularly?

SOP లేకపోతే new joiners struggle అవుతారు.
Without SOPs, new joiners struggle.

Onboarding documentation poor గా ఉంది.
The onboarding documentation is pretty poor.

అది rewrite చేయాలి ఒకసారి.
We need to rewrite it once.

Tickets లో steps document చేయండి.
Document the steps in the tickets.

Future reference కి help అవుతుంది.
It helps for future reference.

Same issue ఈ month లో మూడు సార్లు వచ్చింది.
The same issue came up three times this month.

Root cause document చేస్తే solve అవుతుంది long term.
Documenting the root cause solves it long term.

Wiki maintain చేయాలి up to date.
The wiki needs to be kept up to date.

Search చేస్తే easily find అవ్వాలి answer.
You should easily find the answer when you search.

Documentation effort గా feel అవుతోంది, కానీ pays off.
Documentation feels like effort, but it pays off.

Half hour invest చేస్తే hours of support time save అవుతుంది.
Investing half an hour saves hours of support time later.

Code comments కూడా documentation type.
Code comments are also a form of documentation.

Architecture diagrams ఉండాలి high level కి.
We need architecture diagrams for the high level.

Documentation culture build చేయాలి team లో.
We need to build a documentation culture in the team.
