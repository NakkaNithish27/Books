# 12. Dependencies

ఇది వేరే team మీద depend అవుతోంది.
This depends on another team.

ఆ piece complete అయితే మాత్రమే మనం move అవుతాం.
We can only move once that piece is complete.

ఇది upstream dependency.
This is an upstream dependency.

మన work దీని మీద hooked.
Our work is hooked on this.

ఆ API ready అయితే మనం test start చేస్తాం.
Once that API is ready, we can start testing.

Dependency map ఒకసారి chart చేద్దాం.
Let's chart out the dependency map once.

ఇది వాళ్ళ delivery మీద depend.
This is dependent on their delivery.

మనం వాళ్ళ side నుండి input కోసం wait చేస్తున్నాం.
We're waiting on input from their side.

ఇది critical dependency, miss అవ్వకూడదు.
This is a critical dependency, we can't miss it.

వాళ్ళు delay అయితే మనం కూడా delay.
If they slip, we slip too.

ఈ dependency call out చేయాలి leadership కి.
We need to call out this dependency to leadership.

ఇది parallel గా చేయొచ్చు, dependency లేదు.
This can be done in parallel, no dependency.

ఇది hard dependency, soft కాదు.
This is a hard dependency, not a soft one.

వాళ్ళ ETA మీద మన plan rely అవుతుంది.
Our plan relies on their ETA.

ఈ dependency unblock అయ్యే వరకు hold.
We're on hold until this dependency gets unblocked.

వాళ్ళతో align అయ్యి ఒక common timeline పెట్టుకుందాం.
Let's align with them and lock a common timeline.

ఇది circular dependency గా అవుతోంది.
This is turning into a circular dependency.

వాళ్ళు మన మీద, మనం వాళ్ళ మీద — ఇది deadlock.
They depend on us, we depend on them — it's a deadlock.

మన దగ్గర dependency tracker maintain చేద్దాం.
Let's maintain a dependency tracker on our side.

ఇది dependency గా call out చేస్తాను stand-up లో.
I'll call this out as a dependency in the stand-up.

ఇది external dependency, మన control లో లేదు.
This is an external dependency, not in our control.

వాళ్ళకు heads-up ఇస్తే ముందే plan చేయగలరు.
If we give them a heads-up, they can plan ahead.

ఈ dependency వల్ల timeline risk ఉంది.
There's a timeline risk because of this dependency.

ఎంత time వాళ్ళకి commit చేస్తారో అది మనకి baseline.
Whatever time they commit becomes our baseline.

వాళ్ళు feature freeze చేసాక మాత్రమే మనం start చేస్తాం.
We'll only start once they freeze the feature.

ఈ team మీద ఆ team depend, ఆ team మీద ఇంకొక team.
This team depends on that, and that team on another.

మనం ఈ dependency గురించి ముందే flag చేసాం.
We had flagged this dependency earlier.

వాళ్ళ delivery slip అయితే backup plan ఏంటి?
If their delivery slips, what's our backup plan?

Independent గా ఏం చేయగలమో అది ముందు finish చేద్దాం.
Whatever we can do independently, let's finish first.

ఈ dependency గురించి weekly track చేద్దాం.
Let's track this dependency weekly.
